<objective>
## Goal
Create a `/teddy:cleanup` command that detects and cleans orphaned teams/tasks/worktrees, add STATE.md integrity validation to resume, add pre-flight stale-team checks to apply, and harden unify's cleanup step for partial failures.

## Purpose
When a session dies mid-APPLY or mid-UNIFY, teams, tasks, and worktrees are left orphaned. STATE.md can become inconsistent with the filesystem. Without detection and recovery, the user must manually investigate and clean up before the framework works again. This phase makes Teddy self-healing.

## Output
- New `src/commands/cleanup.md` — detects and resolves orphaned resources
- Modified `apply.md` — pre-flight check warns about stale teams before TeamCreate
- Modified `resume.md` — validates STATE.md integrity against filesystem on resume
- Modified `unify.md` — resilient cleanup that handles partial failures gracefully
- Modified `status.md` — shows health warnings for detected issues
- Modified `teddy.md` — registers cleanup command in entry point
- Modified `bin/install.js` — lists cleanup command in installer output
- New `test/commands.test.js` — structural validation for all command files
</objective>
