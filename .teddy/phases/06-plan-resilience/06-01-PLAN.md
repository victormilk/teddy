
    1. Show what will be reverted:
       `git diff [selected-tag]..HEAD --stat`

       `git log [selected-tag]..HEAD --oneline`
    3. Warn if there are uncommitted changes:
       "WARNING: You have uncommitted changes. Commit or stash them first."
    4. Present confirmation:
       ```
       This will revert [N] commits and [N] file changes.
       STATE.md and ROADMAP.md will be restored to pre-UNIFY state.

       This is DESTRUCTIVE. Proceed? (yes / no)
       ```
    5. **Require explicit "yes" — do not proceed on ambiguous input.**

    **Step 3: execute_rollback**
    1. Create a backup tag of current state: `git tag teddy/pre-rollback/{plan-id}`
    2. Reset to the safety tag: `git reset --hard [selected-tag]`
    3. STATE.md is now restored to its pre-UNIFY state automatically (it was part of the repo at that point)
    4. Verify STATE.md shows loop position at APPLY complete (pre-UNIFY)

    **Step 4: post_rollback**
    1. Report results:
       ```
       ════════════════════════════════════════
       ROLLBACK COMPLETE
       ════════════════════════════════════════

       Reverted to: [tag name]
       Commits undone: [N]
       Backup tag: teddy/pre-rollback/{plan-id}

       STATE.md restored to pre-UNIFY state.
       Loop position: PLAN ✓ → APPLY ✓ → UNIFY ○

       Options:
       [1] Re-run /teddy:unify (retry the merge)
       [2] /teddy:amend-plan (fix the plan first)
       [3] Pause here
       ════════════════════════════════════════
       ```
    2. Do NOT auto-delete the pre-unify tag — user may want to rollback again
    3. Do NOT auto-delete the pre-rollback tag — user may want to undo the rollback

    **IMPORTANT:** Rollback uses `git reset --hard` which is destructive. The backup tag (pre-rollback) provides a safety net but uncommitted changes WILL be lost. Always warn about uncommitted changes first.
  </action>
