# Phase 01 Plan 01: Skill Metadata & Command References

**Adicionar description em cada command e corrigir referências de comando para usar sintaxe com dois-pontos.**

## Objective

- **Goal:** Todos os command files têm `description` no frontmatter YAML e todas as referências de comando usam sintaxe `/teddy:command` (com dois-pontos)
- **Purpose:** Skills aparecem com descrições corretas no listing do Claude Code e rotas NEXT apontam para comandos válidos

## Context

### Descriptions por Command

Cada command file receberá frontmatter YAML com `description` derivada da tag `<purpose>`:

| Command | Description |
|---------|-------------|
| init | Initialize Teddy structure in a new project with .teddy/ directory and project files |
| explore | Analyze project state and codebase to surface opportunities and converge on a direction |
| plan | Create executable PLAN.md with team composition and task assignments for a phase |
| apply | Execute an approved PLAN.md by creating Agent Teams for parallel worktree execution |
| unify | Reconcile plan vs. results, merge worktrees, clean up teams, and close the loop |
| status | Display project status, active team progress, and recommended next action |
| resume | Resume work after session break with context restoration and single next action |
| debug | Systematic debugging with persistent state and optional parallel investigation |
| review | Structured code review of changes, branches, or teammate output |
| map-codebase | Orchestrate parallel Explore agents to analyze and document the entire codebase |

### Formato do Frontmatter

Cada command file receberá este bloco antes da tag `<purpose>`:

```yaml
---
description: <description da tabela acima>
---
```

### Referências a Corrigir

Padrão: `/teddy X` → `/teddy:X` onde X é qualquer nome de comando.

Exemplos:
- `/teddy plan` → `/teddy:plan`
- `/teddy init` → `/teddy:init`
- `` `/teddy apply [plan-path]` `` → `` `/teddy:apply [plan-path]` ``
