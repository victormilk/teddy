## Directory Purposes

**`bin/`:**
- Purpose: NPM installation entry point
- Contains: Single file `install.js` — copies framework to Claude Code config
- Key files: `install.js` (267 lines, Node.js)

**`src/commands/`:**
- Purpose: Slash command handler definitions
- Contains: 11 `.md` files, each defining a `/teddy:{command}` skill
- Key files: `teddy.md` (suite router), `plan.md`, `apply.md`, `unify.md`

**`src/frameworks/`:**
- Purpose: Core methodology documentation referenced by commands
- Contains: 4 `.md` files describing loop phases, team design, plan format, quality principles
- Key files: `loop-phases.md`, `teammate-orchestration.md`

**`src/templates/`:**
- Purpose: Markdown templates used to scaffold output files
- Contains: 6 top-level templates + 7 codebase analysis templates
- Key files: `PLAN.md`, `SUMMARY.md`, `STATE.md`
- Subdirectories: `codebase/` (7 templates for codebase mapping)

**`src/checklists/`:**
- Purpose: Quality validation gates for critical transitions
- Contains: 2 `.md` checklist files
- Key files: `plan-review.md` (pre-APPLY), `unify-gate.md` (pre-UNIFY)

**`src/context/`:**
- Purpose: Runtime session context template loaded by all commands
- Contains: 1 file `session.md`

**`.teddy/`:**
- Purpose: Generated project state (Teddy dogfooding itself)
- Contains: PROJECT.md, ROADMAP.md, STATE.md, phases/, codebase/, debug/
- Generated by: `/teddy:init` and subsequent commands

## Key File Locations

**Entry Points:**
- `bin/install.js` — CLI entry point for `npx teddy-framework`
- `src/commands/teddy.md` — Skill suite definition and command router

**Configuration:**
- `package.json` — npm metadata, version, engines, bin entry
- `.gitignore` — Git exclusions

**Core Logic:**
- `src/commands/plan.md` — Plan creation workflow
- `src/commands/apply.md` — Team spawning and task distribution
- `src/commands/unify.md` — Result reconciliation and merge
- `src/frameworks/teammate-orchestration.md` — Team composition rules

**Documentation:**
- `README.md` — Install instructions, commands overview

## Naming Conventions

**Files:**
- kebab-case for commands: `map-codebase.md`, `loop-phases.md`
- UPPER_SNAKE_CASE for templates/output: `PROJECT.md`, `PLAN.md`, `SUMMARY.md`
- camelCase for JS script: `install.js`

**Directories:**
- kebab-case for source directories: `commands/`, `frameworks/`, `checklists/`
- `{NN}-{descriptive-name}` for phases: `01-skill-metadata/`

**Special Patterns:**
- Phase plans: `{NN}-{PP}-PLAN.md` (e.g., `01-01-PLAN.md`)
- Phase summaries: `{NN}-{PP}-SUMMARY.md`
- Codebase docs: UPPERCASE in `codebase/` subdirectory

## Where to Add New Code

**New Slash Command:**
- Create: `src/commands/{command-name}.md`
- Follow: Existing command format (frontmatter + purpose + steps + acceptance-criteria)
- Register: Add to `src/commands/teddy.md` command list

**New Framework Guide:**
- Create: `src/frameworks/{guide-name}.md`
- Reference from commands via `@frameworks/{guide-name}.md`

**New Template:**
- Create: `src/templates/{DOC-NAME}.md`
- Reference from commands via `@templates/{DOC-NAME}.md`

**New Codebase Template:**
- Create: `src/templates/codebase/{TOPIC}.md`
- Reference from `src/commands/map-codebase.md`

**New Validation Gate:**
- Create: `src/checklists/{gate-name}.md`
- Reference from relevant commands

## Special Directories

**`.teddy/`:**
- Purpose: Generated project state
- Source: Created by `/teddy:init`, populated by commands
- Committed: Yes (tracked in git for dogfooding)

**`.teddy/phases/`:**
- Purpose: Per-phase plans and summaries
- Source: Created by `/teddy:plan` and `/teddy:unify`
- Structure: `{NN}-{name}/{NN}-{PP}-PLAN.md` and `{NN}-{PP}-SUMMARY.md`

**Installation Targets (generated externally):**
- `~/.claude/commands/teddy/` — Global commands (from `src/commands/`)
- `~/.claude/teddy-framework/` — Global support files (frameworks, templates, checklists, context)
- `./.claude/commands/teddy/` — Local commands (project-scoped)
- `./.claude/teddy-framework/` — Local support files
